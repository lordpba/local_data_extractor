📄 Local Data Extractor — Secure Processing Guide

            
            
Environment Compliance & Security

            
This software operates entirely within an offline context to guarantee strict confidentiality. Documents are directly processed by the vision models stored on this hardware, and no data is transmitted to external endpoints.


            
Procedural Guidelines

            
    
                
  • Model Deployment: Verify the local backend model. Ensure a vision-capable Local LLM is selected, sized appropriately for your hardware constraints.
    • 
                
  • Data Ingestion: Securely insert valid document formats (PDF, JPG, PNG). Sensitive documents will be parsed directly in-memory.
    • 
                
  • Data Targets: Utilize compliance presets or configure custom strict JSON schemas to define extraction boundaries.
    • 
                
  • Verification: Upon extraction, manually review the structured payload via the Validation Panel, verifying values for critical items before Excel generation.
    • 
            


            
Execution Strategies

            
    
                
  • Auto Mode — Defers strategy selection to backend logic according to file density.
    • 
                
  • Secure Single Pass — Fast mapping of the document directly via the Vision LLM.
